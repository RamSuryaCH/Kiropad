## 2024-11-20 - Memoizing getLocalIP network lookups
**Learning:** In the Electron app, `getState()` is called frequently by `sendStateToRenderer()`, which is triggered by numerous backend events (pairing changes, lockouts, bridge connections, tunnel states). `getState` subsequently calls `getLocalIP()`. The original `getLocalIP()` performed a synchronous `os.networkInterfaces()` lookup which takes ~0.05ms per call locally, but adds up in tight event loops and can block the main thread.
**Action:** When a synchronous OS-level module (like `os.networkInterfaces()`) is utilized inside frequent getter/state-polling architectures, introduce a short-lived cache (e.g. 30 seconds) to bypass system-call overhead while retaining acceptable freshness. Remember to expose a `clearCache()` for test isolation.
